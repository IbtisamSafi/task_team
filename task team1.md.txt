#### 3.Key Criteria for Writing Clean Code:

 1. Meaningful Names: Use names for variables, functions, and classes that reveal intent and are easily searchable.

 2. Small Functions: Functions must be small (and even smaller!) and perform only one single task at one level of abstraction.

 3. No Redundant Comments: Code should be self-documenting; comments are not a substitute for poorly written code.

 4. Single Responsibility Principle (SRP): Classes should be responsible for one thing only (one reason to change).

 5. Effective Error Handling: Use exceptions instead of return codes, and strictly avoid passing or returning Null.

 6. Clean Unit Tests: Write fast, independent, repeatable, self-validating, and timely (F.I.R.S.T) unit tests.
